## 🧠 عن المشروع

**CyberArsenal** هو موقع ويب تفاعلي (HTML/CSS/JS) صُمم ليكون مرجعًا سريعًا لـ **19 أداة** من أشهر أدوات اختبار الاختراق في توزيعة **Kali Linux**، بالإضافة إلى أدوات خارجية. كل أداة تحتوي على:
- وصف مختصر
- **خطوات التشغيل التفصيلية** (مع إمكانية نسخ الأوامر بنقرة واحدة)
- مستوى الصعوبة (1–3)
- نظام التشغيل المناسب

✨ **الهدف:** توفير مصدر تعليمي أخلاقي للمختبرين الأمنيين والطلاب والمهتمين باختبار الاختراق.

> ⚠️ **تنبيه هام:** هذا المشروع **لأغراض تعليمية وأبحاث أمنية فقط**. المؤلف غير مسؤول عن أي استخدام غير قانوني أو غير أخلاقي لهذه المعلومات.

## 🧰 الأدوات المشروحة (19 أداة)

| القسم | الأداة | الوصف |
|-------|--------|-------|
| 🔑 **قوائم كلمات** | CUPP | توليد كلمات مرور مخصصة من معلومات الهدف |
| 🌐 **خادم ويب** | Apache2 | تشغيل خادم ويب محلي لاستضافة البايلودات |
| 🖼️ **ميتاداتا** | ExifTool | استخراج البيانات المخفية من الصور (GPS، التاريخ، إلخ) |
| 🔓 **كسر كلمات** | John the Ripper (كامل) | كسر PDF، ZIP، Office، Wi-Fi Handshake (مع شروحات متعددة) |
| 🎣 **التصيد الاحتيالي** | PyPhisher, Storm-Breaker, MaxPhisher, Blackeye, Camphish, Nexphisher, Zphisher, Lockphish | إنشاء صفحات مزورة واستغلال الكاميرا والصوت والموقع |
| 📱 **أندرويد** | PhoneSploit-Pro | التحكم بالأندرويد عبر USB → WiFi |
| 📡 **واي فاي** | Aircrack-ng | تخمين كلمة مرور WiFi من ملف handshake |
| 🌍 **استنساخ مواقع** | SiteScraper, WebSift | استنساخ شكل الموقع أو جمع معلوماته |
| 🔐 **ستيغانوغرافي** | StegoForge | إخفاء ملفات داخل صور (txt, exe, mp4, mp3) وتحليل الصور الملغمة |
| 🏹 **قوائم مخصصة** | Cupid | جمع معلومات عن الهدف وإنشاء قاموس خاص |
| 😈 **واي فاي متقدم** | WiFi-Nightmare | Evil Twin باستخدام ESP8266/ESP32 + جلب handshake |

📌 **ملاحظة:** كل أداة تحتوي على خطوات تشغيل مفصلة تظهر عند النقر على بطاقة الأداة.

## 🚀 طريقة الاستخدام (Usage)

### 1. عبر الموقع المباشر
- قم بتحميل ملف `index.html` وفتحه في أي متصفح حديث.
- استخدم **شريط البحث** للعثور على أداة معينة.
- استخدم **أزرار التصفية** لعرض أدوات من نفس الفئة (فيشنج، واي فاي، كسر كلمات، إلخ).
- **انقر على أي بطاقة أداة** لفتح تفاصيل الخطوات.
- **انقر على أي أمر** داخل الخطوات لنسخه تلقائياً إلى الحافظة.

### 2. عبر GitHub Pages (إذا كنت رافع المشروع)
يمكنك رفع المجلد على GitHub واستخدام خاصية Pages لعرض الموقع مباشرة.

### 3. متطلبات التشغيل المحلي
- لا يحتاج إلى خادم ويب (يعمل مباشرة من المتصفح).
- يفضل استخدام **Kali Linux** لتنفيذ الأوامر المذكورة، لكن الموقع نفسه يعمل على أي نظام.
